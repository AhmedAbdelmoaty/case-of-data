

# خطة التعديلات — فهمت كل النقاط، وهنا التفاصيل

## ملخص ما فهمته بالضبط

### 1. صوت طرق الباب 🔊
لما تضغط "اطرق باب المكتب" → عايز صوت طق طق طق **فعلي** يشتغل فوراً.
**الإجابة: أقدر أعمله.** النظام الحالي (`useSoundEffects`) بيولّد أصوات synthesized عبر Web Audio API. فيه بالفعل صوت `"door"` لكنه بيشتغل عند "ادخل المكتب" مش عند الطرق. هضيف نوع صوت جديد `"doorKnock"` — سلسلة 3 نقرات سريعة (3 oscillator bursts) تحاكي طق طق طق — وأشغّله فوراً لما تضغط "اطرق باب المكتب".

### 2. CompanyBriefingScreen — إلغاء مرحلة "office" الوسيطة
**حالياً:** hallway → door-knock → **office** (فيها "مكتب أ. منصور" + زرار "ابدأ المحادثة") → dialogue
**المطلوب:** hallway → door-knock → **dialogue مباشرة** (بدون المرحلة الوسيطة). لما تضغط "ادخل المكتب" → تنتقل مباشرة لصورة mansour-desk مع fade in + المحادثة تبدأ أوتوماتيك بدون أي زرار أو نص.
**التنفيذ:** هحذف phase "office" بالكامل وهخلي "ادخل المكتب" ينقل لـ "dialogue" مباشرة.

### 3. ArrivalScreen — تبسيط المراحل
**حالياً:** storefront → entrance (زرار "ادخل المتجر" + نص) → greeting (نص أبو سعيد + زرار "سلّم عليه") → dialogue
**المطلوب:**
- زرار "اقترب من الباب" → يتغير اسمه لـ **"ادخل المتجر"**
- بعد الضغط → صورة store-entrance تظهر بأنيميشن **تلقائي** لمدة ~2.5 ثانية **بدون أي نص أو زرار** → ثم تنتقل تلقائياً
- بعدها صورة abu-saeed-greeting تظهر بأنيميشن **تلقائي** لمدة ~2.5 ثانية **بدون أي نص أو زرار** → ثم المحادثة تبدأ مباشرة أوتوماتيك
- **كل النصوص والأزرار في entrance و greeting تتشال بالكامل**

### 4. InquiryScreen — حذف نصوص التنقل بين المشاهد
**حالياً:** عند الانتقال لخلفية جديدة (جولة 3 و 4) بيظهر نص:
- "📍 قسم الحريمي" + "أبو سعيد واخدك على قسم تاني..." + "الكلام بيتعمّق أكتر"
- "📍 ركن الكاشير" + "قعدت مع أبو سعيد عند الكاونتر..." + "الكلام بيتعمّق أكتر"

**المطلوب:** حذف **كل** النصوص دي. الانتقال يكون صورة فقط مع transition — بدون أي كلام مكتوب. (لأن النصوص دي تسريب)
**التنفيذ:** هحذف الـ label و subtitle و "الكلام بيتعمّق أكتر" من scene-transition. هيبقي مجرد fade in للخلفية الجديدة لمدة 2-3 ثواني.

### 5. FramingScreen — إصلاح العرض المكرر
**حالياً:** كل بطاقة فيها:
- عنوان bold مقطوع: `option.text.slice(0, 60)...`
- تحته النص الكامل: `option.text`

**المطلوب:** حذف العنوان المقطوع وعرض النص الكامل مرة واحدة فقط بشكل نظيف.

### 6. PresentationScreen — المشكلة الكبيرة 🔴
**فهمت المشكلة بالضبط:** حالياً الكود بياخد نص التأطير المختار وبيلصقه في المحادثة كما هو (بصيغة الغائب مش المخاطب). مثلاً:
- "المبيعات قلّت بسبب المنافسة من المحل اللي **جنبهم**..." — المفروض تكون "**جنبكم**" لأنه بيكلم أبو سعيد
- النص بيتقطع عند 80 حرف فبيبقى ناقص ومش مكتمل

**المطلوب:** كتابة **سكريبت محادثة مخصص لكل اختيار من الـ 4 اختيارات** — كل سكريبت فيه:
- رسالة المحلل بصيغة مخاطبة صحيحة لأبو سعيد (بتقول "عندكم" مش "عندهم")
- النص كامل مش مقطوع
- رد أبو سعيد مخصص لكل اختيار

**التنفيذ:** هكتب 4 سكريبتات محادثة كاملة:

**F1 (الصح — تحويل الاهتمام):** المحلل يشرح لأبو سعيد بشكل مباشر إن المتجر مش عنده مشكلة إقبال، المشكلة في التحويل. أبو سعيد يرد بإعجاب واقتناع.

**F2 (غلط — المنافسة):** المحلل يقول لأبو سعيد إن المنافس اللي **جنبكم** هو اللي بيسحب الزباين. أبو سعيد يرد بشك وعدم اقتناع.

**F3 (غلط — ركود السوق):** المحلل يقول لأبو سعيد إن السوق كله في ركود. أبو سعيد يرد إن السنة اللي فاتت نفس الوقت كان أحسن.

**F4 (غلط — الموظفين):** المحلل يقول لأبو سعيد إن الموظفين مش بيعرفوا يقفلوا البيع. أبو سعيد يرد إن خالد شاطر.

### 7. DebriefScreen — إلغاء مرحلة "establishing"
**حالياً:** hallway (زرار "ادخل المكتب") → establishing (نص "منصور شكله مبسوط..." + زرار "ابدأ التقييم") → dialogue
**المطلوب:** hallway (زرار "ادخل المكتب") → **dialogue مباشرة** بدون المرحلة الوسيطة. لما تضغط "ادخل المكتب" → صورة mansour-desk تظهر + المحادثة تبدأ أوتوماتيك.
**التنفيذ:** هحذف phase "establishing" وهخلي "ادخل المكتب" ينقل لـ "dialogue" مباشرة.

### 8. ReflectionTransition — إضافة صورة خلفية
النص "بتراجع كل اللي سمعته..." و "وبتحاول تصيغ المشكلة الحقيقية" → يفضلوا كما هم. بس هضيف صورة خلفية مناسبة (store-counter.png أو store-inside.png) بدل الخلفية الباهتة الحالية.

## 1. الفلسفة التصميمية

### 1.1 المبادئ الحاكمة

| # | المبدأ | التطبيق |
|---|---|---|
| 1 | **No Leaks** | لا أيقونات/ألوان/تلميحات بصرية تكشف جودة السؤال قبل اختياره |
| 2 | **Cost of Curiosity** | كل سؤال يستهلك من ميزانية محدودة (7 من 20). الاقتصاد الذهني إجباري |
| 3 | **Production over Recognition** | اللاعب يبني التأطير من قطع، لا يختاره من قائمة |
| 4 | **Multiple Plausible Wrong Paths** | 4 فخاخ ذكية. كل فخ يعكس خطأ تفكيري حقيقي |
| 5 | **Earned Insight** | لحظة "آها" تأتي **بعد** التركيب |
| 6 | **Universal Skill** | درس "compared to what?" ينطبق على أي مجال |
| 7 | **Senior-Appropriate** | لغة بسيطة، تحدي ذهني عالي |
| 8 | **Fair Failure** | الفشل من سوء التفكير، لا سوء الفهم |

### 1.2 ما تم إلغاؤه

- ❌ نظام 3 خيارات لكل جولة → قائمة 20 سؤال + ميزانية 7
- ❌ 7 جولات خطية → استقصاء مفتوح بترتيب اللاعب
- ❌ تأطير اختيار من 4 → بناء بالسحب من 3 مكونات
- ❌ نقاط فردية لكل سؤال → تقييم متعدد المحاور في النهاية
- ❌ trustLevel كزخرفة → ثقة ميكانيكية مع عواقب
- ❌ ملاحظات للقراءة فقط → ملاحظات قابلة للسحب في التركيب

## 2. دستور القضية (Case Schema)

```typescript
interface Case {
  id: string;
  title: string;
  industryDomain: string;

  truth: {
    surfaceSymptom: string;
    presentedRequest: string;
    misleadingInterpretation: string;
    hiddenReality: string;
    correctFraming: string;
    keyLesson: string;
  };

  briefingContext: string;
  questionBank: Question[];        // 20
  questionBudget: number;          // 7
  evidencePool: Evidence[];

  framingComponents: {
    symptoms: FramingComponent[];
    rootCauses: FramingComponent[];
    recommendations: FramingComponent[];
  };
  validFramingCombinations: FramingCombo[];

  trustMechanics: {
    initial: number;
    triggers: TrustTrigger[];
  };

  evaluationAxes: EvaluationAxis[];
}

interface Question {
  id: string;
  text: string;                    // بدون أي تلميح للجودة
  category: QuestionCategory;      // داخلي فقط
  reveals: string[];               // evidence IDs
  characterResponse: string;
  characterMood: "neutral" | "confused" | "annoyed" | "thoughtful" | "open";
  trustImpact: number;             // -2 إلى +1
}

type QuestionCategory =
  | "golden"             // 5 — كاشفة للـ baseline
  | "misleading-deep"    // 6 — مضللة بشكل ذكي
  | "premature-solution" // 4 — قفز للحل
  | "neutral-context"    // 3 — معلومات سياقية
  | "frame-challenge"    // 2 — تشكيك في الإطار

interface Evidence {
  id: string;
  text: string;
  weight: "critical" | "supporting" | "contextual" | "distracting";
  pointsTo: string[];
  contradicts: string[];
}

interface FramingCombo {
  symptomId: string;
  rootCauseId: string;
  recommendationId: string;
  tier: "gold" | "silver" | "bronze"
       | "trap-baseline-ignored" | "trap-jumped-to-solution"
       | "trap-internal-cause" | "incoherent";
  feedback: string;
  requiresEvidence: string[];
}
```

## 3. القضية: المقارنة الخاطئة

### 3.1 السياق العام

**العميل**: أبو سعيد، صاحب متجر "بيت الموضة" في منطقة شعبية بالقاهرة. شغّال 12 سنة. عائلي. أبو سعيد في الستينات، خبرة عملية، بيعتمد على إحساسه مش الأرقام.

**ما حدث**: نزل لـ "Pinnacle Consulting" يطلب استشارة. قال إن مبيعات موسم رمضان/العيد **أقل بـ 20%** عن السنة اللي فاتت. عايز "حلّ سريع" — تخفيضات وعروض.

**المهمة (من منصور)**: "روح للمتجر، اتكلم مع أبو سعيد، افهم الموقف، ارجعلي **بتأطير** للمشكلة. مش حلول دلوقتي — تأطير."

### 3.2 العَرَض، التفسير الخاطئ، الحقيقة المخفية

| الطبقة | المحتوى |
|---|---|
| **العَرَض** | مبيعات الموسم الحالي **أقل 20%** من السنة اللي فاتت |
| **الطلب** | "اعمل لي خطة تخفيضات" |
| **التفسير الخاطئ** | "في مشكلة جذب — لازم نحرّك الحركة بالسعر" |
| **الحقيقة المخفية** | الموسم الحالي **طبيعي**. الموسم السابق كان **استثنائياً** بسبب: (أ) افتتاح مول كبير على بُعد 200 متر سبتمبر السنة اللي فاتت — جاب موجة مشاة ضخمة لـ 4-5 شهور. (ب) المنطقة استضافت سوق رمضاني كبير في الميدان قدام المتجر — السنة دي السوق اتنقل |
| **النتيجة لو سار في الطلب** | يخسر 15-20% من هامش الربح على بيع كان هيحصل أصلاً → خسارة مزدوجة |
| **التأطير الذهبي** | "العَرَض مش مشكلة أداء — العَرَض هو **مقارنة غير عادلة**. المرجع كان استثنائياً بسبب عاملين خارجيين انتهوا. الأرقام الحالية هي خط الأساس الطبيعي. **التوصية: لا تخفيضات. أعد بناء baseline من متوسط 3 سنوات**" |

### 3.3 الدرس المكنزي

> **"Any number without a reference point is noise. Before you ask 'why did it drop?', ask 'compared to what?'. The comparison itself is the first hypothesis you must test."**

اللاعب الذكي يدرك:
1. السؤال الأول مش "ليه نزلت؟" بل **"نزلت مقارنة بإيه؟ والمقارنة عادلة؟"**
2. الـ baseline مش حقيقة موضوعية — هو **اختيار** قابل للنقد
3. أحياناً المشكلة الحقيقية إن **المشكلة مش موجودة أصلاً**

### 3.4 الشخصيات

| الشخصية | الوصف | الدور |
|---|---|---|
| **أبو سعيد** | صاحب المتجر، 60s، تاجر بالحس | المصدر الرئيسي. صادق، لكن إجاباته مشوّهة بإطاره الخاطئ |
| **منصور** | مدير الاستشارة | بريفينغ + استقبال + ديبريف |
| **اللاعب** | المحلل | يدير الاستقصاء |

### 3.5 البريفينغ

منصور يعطي **3 معلومات فقط**:
1. اسم العميل وموقع المتجر ومدته
2. العَرَض المُعلن: "20% أقل عن السنة اللي فاتت"
3. الطلب: "عايز خطة تخفيضات"

**ما لا يُعطى** (يُكتشف بالأسئلة):
- الأرقام الفعلية
- كيف قاس الـ 20%
- ما الذي تغير في المنطقة
- نمط المبيعات في السنوات السابقة
- حركة الزباين الفعلية
- هامش الربح وتأثير الخصومات

## 4. بنك الأسئلة الكامل

> **20 سؤال** — اللاعب يختار **7**. كل سؤال **مرة واحدة**. الترتيب لا يؤثر على الإجابات.
>
> **في الواجهة**: بدون أي تصنيف بصري. النظام الداخلي يصنف للحساب فقط.

### 4.1 الأسئلة الذهبية (5)

#### Q1 — التشكيك في المقارنة
**النص**: "السنة اللي فاتت اللي بتقارن بيها — كانت سنة عادية ولا حصل فيها حاجة استثنائية في المنطقة؟"

**أبو سعيد** (مفكر): "استثنائية إزاي يعني؟ ... آه استنى، السنة اللي فاتت كان فيه افتتاح مول جنبنا في سبتمبر، جاب ناس كتير المنطقة. وكمان كان فيه سوق رمضاني كبير في الميدان قدامنا — السنة دي السوق ده اتنقل."

**Reveals**: `E1`, `E2` | **Trust**: +1 | **Category**: golden

---

#### Q2 — البيانات التاريخية
**النص**: "ممكن نشوف أرقام مبيعات الـ 3 أو 4 سنين اللي فاتوا، مش بس السنة اللي فاتت؟"

**أبو سعيد**: "والله أنا مش محتفظ بأرقام تفصيلية، بس عندي إجمالي كل سنة في دفتر. خليني أجيبه ... [يجيب الدفتر] ... بص: 2022 الموسم كان حوالي 180 ألف، 2023 كان 195 ألف، 2024 كان 240 ألف، السنة دي لحد دلوقتي 192 ألف."

**Reveals**: `E3`, `E4` | **Trust**: +1 | **Category**: golden

---

#### Q3 — حركة الزباين الفعلية
**النص**: "الزباين اللي بتدخل المحل دلوقتي — قليلين أو زي العادي؟ وبيشتروا ولا بيتفرجوا؟"

**أبو سعيد**: "والله الحركة... [يفكر] ... مش وحشة. بتدخل ناس عادي. اللي بيدخل أغلبه بيشتري. بس مش زي السنة اللي فاتت اللي كانوا داخلين فوج ورا فوج — السنة دي ناس المنطقة بس."

**Reveals**: `E5`, `E6` | **Trust**: +1 | **Category**: golden

---

#### Q4 — هامش الربح
**النص**: "لو عملنا الخصومات اللي بتفكر فيها، هامش الربح هيتأثر إزاي؟ بتربح كام في القطعة دلوقتي وهتربح كام بعد الخصم؟"

**أبو سعيد**: "الهامش عندي حوالي 35% على القطعة. لو عملت 20% خصم زي ما بفكر، الهامش هيبقى حوالي 15%. ... [يصمت] ... يعني تقريباً نص الربح."

**Reveals**: `E7`, `E8` | **Trust**: 0 | **Category**: golden

---

#### Q5 — التغيرات في المنطقة
**النص**: "في المنطقة حواليك — في حاجة اتغيرت في آخر سنة؟ مولات، أسواق، طرق، أي حاجة؟"

**أبو سعيد**: "آه طبعاً. السوق الرمضاني اللي كان قدامنا اتنقل لميدان تاني. وفي شارع جنبنا قفلوه للترميم من 4 شهور. الحركة في المنطقة كلها قلت شوية مش بس عندي."

**Reveals**: `E2`, `E9`, `E10` | **Trust**: +1 | **Category**: golden

---

#### Q6 — السبب الداخلي: الموظفين

**أبو سعيد**: "لأ والله، نفس الطاقم من 3 سنين. ولاد كويسين، الزباين بتحبهم. مفيش شكاوى."

### 4.5 أسئلة تشكيك في الإطار (2)

#### Q19 — التشكيك في القياس

**أبو سعيد**: "[يفكر] ... بصراحة قعدت قارنت أرقام شهرين الموسم بشهرين موسم السنة اللي فاتت. السنة اللي فاتت كانوا 240 ألف، السنة دي 192. يعني فرق 48 ألف، يعني 20%."

#### Q20 — التشكيك في وجود المشكلة
**النص**: "ممكن نخطو خطوة لورا — هل أكيد إن في مشكلة؟ يعني الأرقام الحالية وحشة فعلاً، ولا بس مش زي السنة اللي فاتت؟"

**أبو سعيد** (مرتبك): "[يصمت] ... يعني ... هي مش وحشة في المطلق. هي بس أقل من السنة اللي فاتت. السنة اللي فاتت كنت مبسوط جداً. بس قبلها ... [يفتح الدفتر] ... قبلها كانت أقل."

**Reveals**: `E22`, `E3` (إشارة) | **Trust**: -1 | **Category**: frame-challenge

### 4.6 ملخص توزيع البنك

| الفئة | العدد | النسبة | الدور |
|---|---|---|---|
| Golden | 5 | 25% | تحل اللغز |
| Misleading-deep | 6 | 30% | فخاخ ذكية |
| Premature-solution | 4 | 20% | فخ القفز للحل |
| Neutral-context | 3 | 15% | مساحة لإهدار الميزانية |
| Frame-challenge | 2 | 10% | الأقوى |
| **الإجمالي** | **20** | **100%** | |

**شرط النجاح**: اكتشاف **3 من 5** أدلة حرجة:
- `E2` (السوق الرمضاني اتنقل)
- `E4` (2024 outlier)
- `E6` (حركة 2024 كانت مضخمة)
- `E10` (تأثير المنطقة كلها)
- `E22` (الأرقام في المطلق ليست سيئة)

دي تنكشف فقط من الذهبية + frame-challenge. مستحيل من الفخاخ.

## 6. نظام التقييم والتقدم

### 6.1 الموارد

| المورد | البداية | الحدود | عواقب |
|---|---|---|---|
| **ميزانية الأسئلة** | 7 | 0-7 | اللعبة تنتهي عند 0 |
| **مستوى الثقة** | 5 | 0-8 | تحت 2: إجابات قصيرة؛ 0: نهاية مبكرة |
| **الأدلة المكتشفة** | 0 | 0-22 | تتبع للتقييم النهائي |

### 6.2 ميكانيكا الثقة (حقيقية، مش زخرفية)

**Triggers:**
- سؤال golden → +1
- سؤال neutral-context → +1
- سؤال misleading-deep → 0 أو -1
- سؤال premature-solution → +1 (خطر! يأكدله إطاره)
- سؤال frame-challenge → 0 أو -1 (مزعج بس قيم)

**Mechanical Effects:**
- **Trust ≥ 6**: أبو سعيد يتطوع بمعلومات إضافية
- **Trust 3-5**: إجابات عادية بطول كامل
- **Trust 2**: إجابات قصيرة، بدون تطوع
- **Trust 1**: "أنا تعبت من الأسئلة، لو عندك حل قوله"
- **Trust 0**: ينهي الجلسة → "Failed Engagement"

### 6.3 محاور التقييم النهائي

| المحور | الوصف | الوزن |
|---|---|---|
| **1. Frame Challenge** | هل سأل سؤال يشكك في الإطار (Q1, Q5, Q19, Q20)؟ | 25% |
| **2. Evidence Discovery** | كم دليل حرج كشف من 5؟ | 25% |
| **3. Question Economy** | نسبة golden/frame-challenge من الـ 7 | 20% |
| **4. Framing Synthesis** | جودة التركيبة (gold/silver/bronze/trap) | 25% |
| **5. Trust Management** | مستوى الثقة النهائي | 5% |

**Tiers:**
- **Master Framer** (90-100): تأطير ذهبي + 4+ أدلة حرجة + سؤال frame-challenge
- **Solid Analyst** (70-89): تأطير فضي + 3 أدلة حرجة
- **Promising** (50-69): تأطير برونزي + بعض الأدلة
- **Missed the Frame** (30-49): تأطير trap + أدلة قليلة
- **Failed** (<30): غير متماسك أو ثقة منهارة

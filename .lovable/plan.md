

# 🎯 خطة نهائية: إعادة هندسة شجرة الأسئلة + حذف Debrief + تنظيف التقارير

## 🗺️ أولاً: إيه هي الـ Tracks؟ (توضيح بسيط)

في الكود الحالي فيه **4 مسارات غلط مُعرّفة** (A, B, C, D) — كل واحد بياخد اللاعب في اتجاه تشخيصي مختلف:

| المسار | التشخيص الغلط | البيانات اللي بيشوفها اللاعب |
|---|---|---|
| **TRACK_A** | المشكلة في فريق البيع (موظفين مش شغّالين) | كميات بيع لكل موظف + نسب تحويل |
| **TRACK_B** | ❌ *مكرر/ضعيف — مش متربط بإطار نهائي واضح* | (هيتشال — مفيش له lookup في framing-board) |
| **TRACK_C** | المشكلة في المنافسين والأسعار | عروض المنافسين + مقارنة أسعار |
| **TRACK_D** | المشكلة في التسويق/الحملات | أداء الحملات الإعلانية + الميزانية |

✅ **هنبقى بـ 3 مسارات غلط بس: A, C, D** (هنشيل B لأنه مش مربوط بإطار نهائي ومالوش معنى تشخيصي مستقل).

## 🧭 ثانياً: تصميم الشجرة الجديدة بالكامل

### 📌 قواعد ثابتة
- ✅ **المسار الصحيح (S1→S2→S3→S4→S5) مش هيتغيّر فيه ولا حرف.**
- ✅ **مفيش Recovery (R1) خالص** — أي اختيار غلط يدخل مسار غلط مباشرة.
- ✅ كل مسار غلط بياخد 3 أسئلة جوّاه قبل ما يخلص → هشام بيقتنع → اللاعب يطلع للتأطير → منصور هو اللي يكشف الغلط لاحقاً.

### 🌳 خريطة الشجرة الكاملة

```text
                          [S1: الإحساس جالك إزاي؟]
                              /              \
                          صح                  غلط ("حاسس فريق البيع")
                          /                      \
              [S2: قلت بنسبة كام؟]          [TRACK_A_1] ──→ [TRACK_A_2] ──→ [TRACK_A_3] ──→ END
                  /        \                  (مسار فريق البيع — كامل ومتماسك)
              صح            غلط
              /               \
   [S3: ممكن نشوف        [S2_WRONG: تقرير المبيعات اليومي]
    الأرقام على ورق؟]              ↓
       /        \         (هشام يوريه التقرير — مبيعات الأسبوع الأخير ضعيفة)
   صح            غلط               ↓
   /              \         [BRIDGE_D: "طب الفترة دي في حملات شغّالة؟"]
[S4: تقرير 3      [S3_WRONG:        ↓
 سنين؟]            "في حملة         [TRACK_D_2] ──→ [TRACK_D_3] ──→ END
   /    \           ترويج؟"]        (مسار التسويق — كامل)
  صح     غلط          ↓
  /       \      [TRACK_C_1] ──→ [TRACK_C_2] ──→ [TRACK_C_3] ──→ END
[S5]    [TRACK_C_*]  (مسار المنافسين — كامل)
 / \
صح  غلط
/    \
END  [TRACK_D_*]
```

### 🔑 نقاط الدخول لكل مسار غلط

| المسار | يفتح من |
|---|---|
| **A (فريق البيع)** | S1 wrong فقط — يكمل لآخره (3 أسئلة جوّا A). |
| **D (التسويق)** | S2 wrong (عبر تقرير المبيعات اليومي → سؤال جسر) **أو** S5 wrong |
| **C (المنافسين)** | S3 wrong (سؤال جديد عن الترويج) **أو** S4 wrong |

كل مسار غلط له **نقطة دخول مبكرة** (S1 أو S2 أو S3) و(C, D) عندهم نقطة دخول تانية متأخرة كمان — يعني اللاعب ممكن يدخلهم من كذا طريق.

## 📜 ثالثاً: التغييرات في `case-tree.ts` بالتفصيل

### 1. **S1** — تعديل وجهة الغلط فقط
- `wrong.text` و `hishamReply`: **زي ما هما** ("حاسس المشكلة من فريق البيع").
- `nextOnWrong: "TRACK_A_1"` *(بدل R1)*

### 2. **S2 wrong** — يفضل زي ما هو + يدخل سؤال جسر جديد
- النص: زي ما هو ("ممكن أبص على تقرير المبيعات الشهر ده؟").
- `hishamReply`: زي ما هو (هشام يديه التقرير اليومي — يلاحظ ضعف الأسبوع الأخير).
- `evidenceId: "ev_weekly_sales"` *(زي ما هو)*.
- `nextOnWrong: "BRIDGE_D"` *(بدل TRACK_B_1)*.

### 3. **BRIDGE_D** — Node جديد (سؤال جسر للتسويق)
```ts
BRIDGE_D: {
  id: "BRIDGE_D",
  correct: {
    text: "شفت الأسبوع الأخير ضعيف. طب في حملات إعلانية أو ترويج شغّال الفترة دي؟",
    hishamReply: "والله الحملات قليلة الفترة دي. كنا بنعتمد على الزبون الدائم، بس الجدد قلوا.",
    evidenceId: "ev_marketing_spend",
    note: { id: "n_bd", text: "هشام أكد إن الحملات قلّت — اللاعب بدأ يربط الضعف بالتسويق." },
  },
  wrong: { /* نفس الـ correct — مفيش اختيار حقيقي هنا، سؤال انتقالي */ },
  nextOnCorrect: "TRACK_D_2",
  nextOnWrong: "TRACK_D_2",
}
```
*(الـ BRIDGE هو سؤال واحد بيظهر للاعب كاختيار وحيد — انتقال طبيعي بعد ما شاف التقرير.)*

### 4. **S3 wrong** — جديد بالكامل (يفتح TRACK_C)
- `correct`: زي ما هو ("ممكن نشوف الأرقام دي على ورق؟").
- `wrong` جديد:
  ```ts
  text: "طب قبل ما نكمل، في حملات ترويج شغّالة الفترة دي؟ ممكن السوق محتاج تذكير."
  hishamReply: "حملاتنا قليلة. والمنافسين حواليّا بيعملوا عروض كل أسبوع — ممكن ده اللي بيسحب الزباين."
  evidenceId: "ev_competitor_offers"
  note: "اللاعب قفز للمنافسين قبل ما يتأكد من أصل المقارنة."
  ```
- `nextOnWrong: "TRACK_C_1"`

### 5. **S4** و **S5** — مفيش تغيير في النصوص
- S4 wrong → `TRACK_C_1` *(زي ما هو)*.
- S5 wrong → `TRACK_D_1` → نخليه يروح `TRACK_D_2` مباشرة *(عشان TRACK_D_1 هيتشال ويتعوّض بـ BRIDGE_D)*.

### 6. **TRACK_A** (3 أسئلة) — مفيش تغيير في المحتوى
A_1 → A_2 → A_3 → END. أسئلة عن المندوبين والتحويل والساعات.

### 7. **TRACK_C** (3 أسئلة) — مفيش تغيير في المحتوى
C_1 → C_2 → C_3 → END. أسئلة عن أسعار المنافسين والعروض والموقع.

### 8. **TRACK_D** — تعديل بسيط
- نشيل **TRACK_D_1** (لأن BRIDGE_D بقى نقطة الدخول).
- نخلي **TRACK_D_2** و**TRACK_D_3** زي ما هما.
- النقطة دي مهمة: مفيش فقد محتوى، الـ D_1 كان سؤال افتتاحي عام، البريدج بقى أحسن منه.

### 9. **TRACK_B** — يتشال بالكامل
نشيله من `NodeId` union ومن `NODES` map. مفيش node بيشاور عليه بعد التغييرات.

### 10. **R1** — يتشال بالكامل
نشيل الـ recovery من الكود ومن state machine.
